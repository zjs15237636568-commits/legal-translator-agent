> Version: V1.0-MVP  
> Status: 已评审，开工基线

---

## 0. 最终决策摘要

| 项目 | 决策 |
|---|---|
| 部署 | 前后端本地化；LLM 走公网 API |
| LLM 供应商 | **OpenAI / Claude / DeepSeek** 三选一，运行时可切换 |
| 后端 | Python 3.10+ / **FastAPI** / SSE |
| 前端 | **Vue 3 + Vite + Tailwind + Pinia** |
| 解析 | `python-docx`（Word） + `PyMuPDF`（PDF） |
| 存储 | **SQLite**（历史/项目），`glossary.json`（术语），内存处理文件不落盘 |
| 分段 | Clause/Section → 段落 → Token 三级，相邻 200 token overlap |
| 对齐 | 段落级 `data-anchor-id` 锚点 |
| 术语库 | **仅限专有名词**，全词匹配（大小写不敏感），前端 CRUD，扁平结构 |
| 风险引擎 | **纯 LLM 判定**（MVP 不做规则库） |
| 风险时机 | 翻译完成后一次性扫描 |
| 版本比对 | 英文 vs 英文；LLM 语义 diff + difflib 定位 |
| 导出 | 中英对照 Word（双栏表格）+ 风险批注 |
| 鉴权 | 单机单用户，无鉴权；API Key AES 加密落盘 |
| 单文件上限 | 20MB |
| 并发 | 段落内部并发 = 3 |

## 2. 数据模型 (SQLite)

```sql
-- 项目（一次上传 = 一个项目）
CREATE TABLE projects (
  id             TEXT PRIMARY KEY,          -- uuid
  name           TEXT NOT NULL,
  source_name    TEXT,                      -- 原文件名
  source_type    TEXT,                      -- docx / pdf
  created_at     INTEGER,
  updated_at     INTEGER,
  llm_provider   TEXT,                      -- openai/claude/deepseek
  llm_model      TEXT,
  status         TEXT                       -- parsing/translating/done/failed
);

-- 段落（Clause/段落级）
CREATE TABLE segments (
  id             TEXT PRIMARY KEY,          -- 项目内递增锚点 ID
  project_id     TEXT,
  seq            INTEGER,                   -- 顺序
  clause_no      TEXT,                      -- "3.1" / NULL
  heading        TEXT,                      -- 标题（可空）
  original_md    TEXT,                      -- 原文 Markdown
  token_count    INTEGER,
  FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 翻译结果（支持断点续翻 + 重试）
CREATE TABLE translations (
  segment_id     TEXT PRIMARY KEY,
  translated_md  TEXT,
  status         TEXT,                      -- pending/streaming/done/failed
  error          TEXT,
  retry_count    INTEGER DEFAULT 0,
  updated_at     INTEGER
);

-- 风险项
CREATE TABLE risks (
  id             TEXT PRIMARY KEY,
  project_id     TEXT,
  segment_id     TEXT,                      -- 关联段落（用于跳转）
  level          TEXT,                      -- red / yellow
  category       TEXT,                      -- 违约/数据合规/管辖/…（LLM自由归类）
  title          TEXT,
  detail         TEXT,
  suggestion     TEXT,
  created_at     INTEGER
);

-- Q&A 会话
CREATE TABLE qa_messages (
  id             TEXT PRIMARY KEY,
  project_id     TEXT,
  role           TEXT,                      -- user / assistant
  content        TEXT,
  citations      TEXT,                      -- JSON: [{segment_id, clause_no}]
  created_at     INTEGER
);
```

**术语库** 独立于 DB：`./data/glossary.json`

```json
[
  { "en": "Official Wallet", "zh": "官方钱包（作为具备收、付、兑能力的底层容器，非独立实体）", "note": "" }
]
```

**配置** `./data/config.enc`（AES-GCM）：

```json
{
  "active_provider": "deepseek",
  "providers": {
    "openai":   { "api_key": "...", "base_url": "https://api.openai.com/v1", "model": "gpt-4o" },
    "claude":   { "api_key": "...", "base_url": "https://api.anthropic.com", "model": "claude-sonnet-4" },
    "deepseek": { "api_key": "...", "base_url": "https://api.deepseek.com", "model": "deepseek-chat" }
  }
}
```

## 3. 核心流程

### 3.1 上传 → 翻译（SSE）

```
POST /api/projects/upload   (multipart)
  └─► 解析 → 分段 → 入库 → 返回 project_id + segments 列表

GET  /api/projects/{id}/translate/stream   (SSE)
  └─► 事件流：
      event: segment_start  data: {segment_id}
      event: segment_delta  data: {segment_id, delta}
      event: segment_done   data: {segment_id, full_text}
      event: segment_error  data: {segment_id, error}
      event: all_done       data: {total, duration_ms}
```

### 3.2 翻译完成 → 风险扫描

```
POST /api/projects/{id}/risks/scan
  └─► 分批把段落投喂 LLM，要求返回结构化 JSON：
      [{segment_id, level, category, title, detail, suggestion}]
  └─► 入库 + 返回
```

### 3.3 Q&A（RAG）

```
POST /api/projects/{id}/qa
  body: { question }
  ── 检索：对段落做轻量向量化（OpenAI embedding 或 BM25 兜底）
  ── 组装 Prompt（top-k segments + 强制引用格式）
  ── 返回：{ answer, citations: [{segment_id, clause_no}] }
```

> MVP 阶段检索：**BM25**（`rank_bm25`）即可，零依赖、零成本，后续按需升级向量。

### 3.4 版本比对

```
POST /api/diff
  body: { left_project_id, right_project_id }
  ── 段落级对齐（LLM 判定语义相近段落配对）
  ── 对配对段落调用 LLM 产出：
      { added / removed / modified, business_impact }
  ── 前端按章节渲染 diff
```

## 4. 关键模块设计

### 4.1 LLM Router（统一抽象）

```python
class LLMProvider(Protocol):
    async def stream_chat(self, messages, **kwargs) -> AsyncIterator[str]: ...
    async def chat(self, messages, **kwargs) -> str: ...
    async def embed(self, texts: list[str]) -> list[list[float]]: ...  # 可选

class OpenAIProvider(LLMProvider): ...
class ClaudeProvider(LLMProvider): ...
class DeepSeekProvider(LLMProvider): ...   # OpenAI 兼容协议，复用 OpenAIProvider 仅改 base_url

def get_provider() -> LLMProvider:
    cfg = load_config()
    return REGISTRY[cfg.active_provider](cfg)
```

> DeepSeek API 与 OpenAI 完全兼容，直接用 `openai` SDK + 替换 `base_url=https://api.deepseek.com` 即可，**不额外增加 SDK**。

### 4.2 Segmenter（三级切分）

```
1. 按 Markdown 标题（#, ##, ###）切 Section
2. 每个 Section 内按空行切段落
3. 单段 > MAX_TOKEN(1500) 时按句子切，相邻块保留 200 token overlap
4. 识别 clause_no：正则 ^(\d+(\.\d+)*)\s+ 或 Article/Clause 前缀
```

### 4.3 Glossary Matcher

- 加载 `glossary.json` → 构造 `re.compile(r'\b(term1|term2|...)\b', re.IGNORECASE)`
- 对段落扫描 → 命中的词对作为 Prompt 变量注入
- **不做语境判断**（决策：交给大模型）

### 4.4 Translation Prompt（System）

```
You are a senior cross-border legal counsel translator.
Target language: Simplified Chinese.
Principles:
- Preserve Markdown structure, headings, lists, tables, and inline emphasis.
- Maintain legal rigor: no omission, no paraphrasing of obligations.
- Use formal Chinese legal register (e.g., "应当", "不得", "经…同意").
- Keep clause numbers and cross-references verbatim.

Fixed glossary (MUST use exactly):
{glossary_pairs}

Return ONLY the translated Markdown. No commentary.
```

### 4.5 Risk Prompt（全文一次性）

```
You are a cross-border legal + business risk reviewer.
Scan the following contract segments and return a JSON array.

Risk levels:
- "red":    legal breach / invalid clause / regulatory non-compliance
- "yellow": business feasibility concerns (SLA, settlement timing, system capacity, data flow)

Each item MUST include: segment_id, level, category, title, detail, suggestion.
Return strict JSON only.
```

### 4.6 Q&A Prompt（强制引用）

```
Answer the user's question based ONLY on the provided segments.
Every factual claim MUST be followed by a citation like [Clause 3.1] or [§12].
If the answer is not in the segments, say so explicitly.
```

### 5.1 段落级同步滚动算法

```ts
// 左右面板每个段落包 <div data-anchor-id="seg_xxx">
// 监听左面板 scroll：
//   1. 找到 viewport 顶部最接近的 segment
//   2. 计算该 segment 在左侧的相对偏移 ratio
//   3. 在右侧找到同 id 段落，scrollTo 到对应位置（带 ratio 微调）
// 加 debounce(16ms) + 互斥锁避免循环触发
```

### 5.2 引用点击跳转

```ts
// Q&A 回答里 [Clause 3.1] 被渲染成 <a data-cite="seg_xxx">
// click → document.querySelector(`[data-anchor-id="${id}"]`).scrollIntoView({behavior:'smooth', block:'center'})
// 同时高亮 2 秒
```

## 7. 非功能性

| 项 | 值 |
|---|---|
| 单文件上限 | 20MB |
| 单次 LLM 调用超时 | 120s |
| 段落并发 | 3 |
| Rate limit 退避 | 指数退避，最多重试 3 次 |
| SSE 心跳 | 15s |
| 失败段 | 标红，支持单段重试 `POST /api/segments/{id}/retry` |
| 敏感信息 | API Key AES-GCM 加密落盘；原文件不落盘 |
